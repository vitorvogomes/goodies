# Goodies — UX & Design Spec

> Produzido por Sally (UX/BMAD) + Aspásia (persuasão e copy financeiro).
> Input primário para EPIC-07 e para qualquer decisão de componente de UI nos épicos anteriores.
> Tela mínima: 1024px (desktop-first). Dark mode exclusivo no MVP.

---

## 1. Princípios de design

Estes princípios guiam toda decisão visual e de interação. Quando houver dúvida, volte aqui.

**1. Dados têm contexto ou não têm valor.**
XIRR de 8,5% sozinho é inútil — ao lado do CDI de 10,8% ele conta uma história. Nenhum número aparece sem referência ou benchmark. (Aspásia: um investidor sem benchmark está voando às cegas — o Goodies resolve isso visualmente.)

**2. Ação antes de informação.**
O dashboard não é um relatório. É uma superfície que responde "o que eu faço agora?". A hierarquia visual prioriza o que requer decisão (alertas, desvio de alocação) sobre o que é apenas monitoramento (histórico de transações).

**3. Negativo não significa ruim, mas precisa de contexto.**
Cripto em -18,8% no período pode ser esperado dado o ciclo. O sistema comunica a posição sem alarmismo gratuito — mas sem esconder a realidade. Vermelho é vermelho, não laranja.

**4. Latência de dado é latência de confiança.**
Dado desatualizado é mais perigoso que dado ausente. Todo valor com mais de 2h de idade exibe um indicador visual de "stale" — sutil mas presente.

**5. Um usuário, zero ruído.**
Sem notificações que não pedem ação. Sem tooltips em dados óbvios. Sem animações ornamentais. Cada pixel justifica sua presença.

## 2. Arquitetura de informação

### Hierarquia de navegação

```
Goodies
├── Dashboard (home — visão consolidada)
├── Ledger
│   ├── Transações (lista + filtros)
│   ├── Resumo mensal
│   └── Projeção de caixa
├── Portfolio
│   ├── Posições (visão atual)
│   ├── Alocação + Rebalanceamento
│   └── Operações (histórico)
├── Mercado
│   ├── Preços atuais
│   └── Wallets
└── Analytics
    ├── Retorno (XIRR vs benchmarks)
    ├── Projeções
    └── Metas
```

**Alertas:** não têm página própria. Aparecem no Dashboard (card de prioridade) e como badge no ícone de sino no header. Dispensados ou resolvidos desaparecem silenciosamente.

### Profundidade máxima: 2 níveis
Nenhuma feature está a mais de 2 cliques do dashboard. Se o usuário não encontra algo em 2 cliques, é um problema de IA, não de feature.

## 3. Sistema de design

### 3.1 Paleta de cores

**Fundo e superfícies (dark)**

| Token | Hex | Uso |
|---|---|---|
| `bg-base` | `#09090E` | Fundo da página |
| `bg-surface` | `#111118` | Cards, painéis |
| `bg-elevated` | `#1A1A24` | Modais, dropdowns |
| `bg-hover` | `#1E1E2E` | Estado hover de rows |
| `border-subtle` | `#22223A` | Bordas de cards |
| `border-default` | `#2E2E4A` | Bordas de inputs |

**Texto**

| Token | Hex | Uso |
|---|---|---|
| `text-primary` | `#E8E8F2` | Títulos, valores principais |
| `text-secondary` | `#8A8AAE` | Labels, legendas |
| `text-tertiary` | `#55557A` | Metadados, timestamps |
| `text-disabled` | `#33334A` | Campos desabilitados |

**Cores semânticas financeiras**

| Token | Hex | Quando usar |
|---|---|---|
| `gain` | `#22C55E` | Retorno positivo, acima do alvo, receita |
| `loss` | `#EF4444` | Retorno negativo, abaixo do esperado |
| `warning` | `#F59E0B` | Alertas de atenção, dado stale, IR próximo do limite |
| `critical` | `#DC2626` | Alertas críticos (concentração Flash), risco imediato |
| `info` | `#3B82F6` | Informação neutra, seleção |
| `accent` | `#7C3AED` | XIRR e Analytics (destaque analítico — violeta) |
| `neutral` | `#6B7280` | Dados sem valência (volume, timestamps recentes) |

**Regra do vermelho:** vermelho (`loss` e `critical`) é reservado para perda ou risco real. Desvio de alocação é `warning` (amarelo), não `loss` — ele é oportunidade de aporte, não falha.

### 3.2 Tipografia

| Uso | Font | Weight | Size |
|---|---|---|---|
| Títulos de página | Inter | 600 | 24px |
| Títulos de card | Inter | 500 | 16px |
| Labels | Inter | 400 | 14px |
| **Valores numéricos** | **JetBrains Mono** | **500** | **16–24px** |
| Valores secundários | JetBrains Mono | 400 | 13px |
| Metadados/timestamps | Inter | 400 | 12px |

**Por que monospace para números:** alinhamento de colunas em tabelas, dígitos de largura constante (sem "1" mais estreito que "8"), padrão visual de precision que comunica seriedade financeira.

Instalar via `next/font`:
```tsx
import { Inter, JetBrains_Mono } from 'next/font/google'
const inter = Inter({ subsets: ['latin'], variable: '--font-sans' })
const mono = JetBrains_Mono({ subsets: ['latin'], variable: '--font-mono' })
```

### 3.3 Espaçamento e grid

- Grid: 12 colunas com gap de 24px
- Padding de card: 20px
- Padding de página: 24px (lateral) + 32px (topo)
- Border radius de card: 12px
- Border radius de badge/pill: 6px
- Sidebar width: 240px (collapsed: 64px — ícone apenas)

### 3.4 Componentes base

**Value badge** — o átomo mais usado
```
┌────────────────────────────┐
│  +8,53%   ↑ acima do CDI  │  ← cor semântica no texto e borda esquerda
└────────────────────────────┘
```

**Staleness indicator** — dado desatualizado
```
• PETR4   R$ 38,40   [⚠ 6h atrás]
```
O badge `[⚠ 6h atrás]` aparece em `warning` quando > TTL + 30min. Hover mostra timestamp exato.

**Alert card** — hierarquia de severidade
```
╔══════════════════════════════════════════════════════╗
║ 🔴 CRÍTICO  Concentração Flash                       ║
║ Flash é 71% da sua renda E 43% da sua Renda Fixa.   ║
║ Um problema lá = dois problemas aqui.               ║
║                          [Entender →]  [Dispensar]  ║
╚══════════════════════════════════════════════════════╝
```

**Desvio de alocação** — o principal driver de decisão de aporte
```
CRIPTO     ████████████████░░░░  13,7%  (meta 5,0%)  +8,7pp  ⚠
RF         ████████████████████████████  47,6%  (meta 50,0%)  -2,4pp  →
```
Barra preenchida = atual. Barra tracejada = meta. Texto `+Xpp` em `warning`, `-Xpp` em cor neutra (oportunidade de aporte, não problema).

### 5.3 Formatação de números (utilitários)

```typescript
// lib/format.ts

// Moeda em BRL
export const formatBRL = (value: number): string =>
  new Intl.NumberFormat('pt-BR', { style: 'currency', currency: 'BRL' }).format(value)
// R$ 28.303,02

// Percentual com sinal
export const formatPercent = (value: number, decimals = 2): string => {
  const sign = value > 0 ? '+' : ''
  return `${sign}${value.toFixed(decimals)}%`
}
// +8,53%  |  −1,31%

// XIRR anualizado (decimal → percentual)
export const formatXIRR = (rate: number): string =>
  formatPercent(rate * 100)

// Quantidade de ativos (respeita casas decimais relevantes)
export const formatQuantity = (qty: number, ticker: string): string => {
  const isCrypto = ['BTC', 'ETH', 'SOL'].includes(ticker)
  return isCrypto ? qty.toFixed(6) : qty.toFixed(0)
}

// Data em formato BR
export const formatDate = (date: Date): string =>
  new Intl.DateTimeFormat('pt-BR').format(date)
// 02/06/2026

// "há N horas" para staleness
export const formatStaleness = (lastUpdated: Date): string => {
  const hours = Math.floor((Date.now() - lastUpdated.getTime()) / 3600000)
  if (hours < 1) return 'há menos de 1h'
  if (hours === 1) return 'há 1h'
  return `há ${hours}h`
}
```

### 6.3 Atualização de preço manual

Click no badge `[⚠ preço manual — atualizar]` abre inline editor na própria row (não drawer):

```
Flash D.  [13.207,62     ] R$  ✓  ✗
```

Enter ou ✓ salva. Esc cancela. Foco retorna para a tabela.

### 6.4 Estados de loading

**Skeleton loading** (não spinner) para carregamentos iniciais de dados. O layout é preservado durante o load — o usuário sabe onde os dados vão aparecer.

```
┌──────────────────────────────┐
│  ████████████████  24px      │  ← título skeleton
│  ██████████  16px            │  ← valor skeleton
└──────────────────────────────┘
```

Animação: `animate-pulse` Tailwind. Duration: 1,5s. Sem bounce.

**Para atualizações em background** (refresh de preços): não bloquear a UI. Exibir spinner de 16px no ícone de ↻ no header da seção de mercado. O dado antigo permanece visível até o novo chegar.

### 6.5 Empty states

Cada engine tem um empty state específico — não "Nenhum dado encontrado" genérico.

| Tela | Empty state |
|---|---|
| Ledger/transações | "Nenhuma transação ainda. Registre a primeira com N." |
| Portfolio/posições | "Nenhuma operação registrada. Comece registrando sua primeira compra com O." |
| Alertas | "Nenhum alerta ativo — tudo dentro dos parâmetros. 👌" |
| Wallets | "Wallets ainda não escaneadas. O próximo scan automático ocorre às 20h." |

**Nota de copy (Aspásia):** empty state de alertas usa tom positivo com emoji sutil. Vazio aqui é uma boa notícia, não uma falha do sistema.

### 6.6 Confirmações e destrutivas

**Sem confirmação para:** edições de valor, marcação de alerta como lido, atualização de preço manual.

**Com confirmação modal para:** exclusão de transação, exclusão de operação de investimento, exclusão de custo fixo.

Modal de confirmação:
```
┌───────────────────────────────────────┐
│  Excluir transação?                   │
│                                       │
│  Aluguel — R$ 1.200 — 01/06/2026     │
│  Esta ação não pode ser desfeita.     │
│                                       │
│  [Cancelar]     [Excluir]             │
└───────────────────────────────────────┘
```

`[Excluir]` em `loss` (vermelho). `[Cancelar]` em `bg-elevated` (neutro). Foco inicia em `[Cancelar]` — a ação segura é o default.

## 7. Copy e linguagem (Aspásia)

### 7.1 Tom geral

O Goodies fala com um investidor que sabe o que está fazendo. Sem tutoriais desnecessários, sem explicações de termos que ele já domina. O tom é o de um sistema de controle profissional — objetivo, preciso, sem condescendência.

Mas cuidado: objetivo não é frio. Quando algo é positivo, o sistema reconhece. Quando é preocupante, diz claramente — sem dramatizar.

### 7.2 Glossário de copy

| Situação | ❌ NÃO usar | ✅ Usar |
|---|---|---|
| Retorno da carteira | "Rentabilidade", "ganho" | "Retorno (XIRR)" — sempre com metodologia |
| Desvio de alocação | "Está errado", "fora do alvo" | "X,Xpp acima/abaixo do alvo" |
| Dado desatualizado | "Erro", "falha" | "Último valor de [timestamp] — aguardando atualização" |
| IR cripto próximo | "CUIDADO!" | "Vendas de cripto em [mês]: R$ XX.XXX — R$ X.XXX do limite de isenção" |
| Cripto em perda | "Prejuízo" | "−18,8% no período" |
| Concentração Flash | "Risco de concentração" | "Flash é empregador + maior ativo de RF. Um problema lá = dois problemas aqui." |
| Meta de LF | "Muito longe ainda" | "22 anos ao ritmo atual. Com aporte de R$ 4.500/mês, cai para 14 anos." |

### 7.3 Alertas — copy por tipo

**Rebalanceamento:**
> "RF está 2,4pp abaixo do alvo. No próximo aporte de R$ 4.500, sugerimos R$ 1.850 para RF."

**Concentração Flash:**
> "Flash Capital representa 71% da sua renda mensal e 43% da sua Renda Fixa. Um evento adverso na Flash afeta renda e patrimônio simultaneamente — monitorar."

**IR cripto:**
> "Você vendeu R$ 31.200 em cripto em junho/2026 — 89% do limite de isenção (R$ 35.000). Próximas vendas neste mês podem gerar obrigação tributária."

**Vencimento DeFi (30 dias):**
> "xSOL e hyUSD (Phantom/Hylo) vencem em 29/09/2026 — 30 dias. Definir o que fazer com as posições."

**Vencimento DeFi (7 dias):**
> "xSOL e hyUSD vencem em 7 dias (29/09/2026). Ação necessária."

**Nota:** alertas de 7 dias sobem de `warning` para `critical`. Copy muda de "definir o que fazer" para "ação necessária" — a urgência aumenta com a proximidade.

### 7.4 Labels de números

Sempre que exibir XIRR pela primeira vez em uma tela, incluir o badge de metodologia:
```
XIRR: +8,53% a.a.  [ℹ metodologia DCA-correta]
```
Hover no ℹ mostra: "Calculado via taxa interna de retorno estendida, considerando o timing de cada aporte. Mais preciso que retorno simples para portfólios com aportes regulares."

Isso está lá não por tutoriais — está lá porque o usuário vai questionar "por que XIRR é diferente dos 3,14% da planilha". O badge antecipa essa pergunta sem poluir o layout.

## 8. Acessibilidade

Escopo mínimo para MVP (single-user que conhece o sistema):

- Contraste: todos os textos em `text-primary` sobre `bg-base` passam WCAG AA (ratio > 4,5:1)
- Navegação por teclado: todos os formulários e ações principais acessíveis via Tab + Enter
- Foco visível: `outline: 2px solid #7C3AED` (accent) em todos os elementos focáveis — não remover outline
- Tabelas: `<th>` com `scope="col"`, valores numéricos em `<td aria-label="valor: R$ 28.303">`
- Alertas: `role="alert"` para alertas críticos (lidos pelo screen reader imediatamente)
- Cores não são o único indicador: ícones (↑↓→) acompanham cores semânticas

## 9. Implementação — libs e decisões

| Decisão | Escolha | Alternativa descartada | Por quê |
|---|---|---|---|
| Charts | Recharts | Chart.js, Victory | API declarativa (JSX), melhor integração com React |
| Componentes base | Radix UI primitives | shadcn/ui | Menos footprint, mais controle sobre estilo |
| Ícones | Lucide React | Heroicons | Consistência com design de linha fina |
| Fontes | Inter + JetBrains Mono | Sistema | Monospace para números é inegociável |
| Animações | Tailwind animate | Framer Motion | Framer Motion é overkill para animações de loading |
| Form state | React state nativo | React Hook Form | Formulários são simples (max 6 campos) |
| Server state | TanStack Query v5 | SWR | Mais controle, melhor DevTools |
